# KennelFlow Core 0.3.23

Since WordPress.org **0.2.3** (GitHub-only releases in between).

## Hub calendar

- Shared React calendar for boarding, grooming, and clinic with drag-and-drop, timezone-aware labels, and booking popovers.
- **Check in** on appointment popovers; **Add walk-in** on clinic calendars.
- Popover links: Edit/View appointment, Edit pet, Patient history, Edit owner (capability-aware).
- **Booking session photos** in popovers for all spoke types (grooming before/after, boarding check-in/out, clinic intake/discharge) when the spoke plugin registers settings.
- Live device camera capture (**Take photo**) plus **Choose photo** from files.
- Front-end `[ltkf_hub_calendar]` with `booking_kind` / `corner_label`; spoke-specific calendar caps via `ltkf_user_can_view_hub_calendar`.
- Calendar debug panel, error boundary, and improved Add booking modal (portal, pet owner labels, boarding dependency notices).

## Owner portal & booking

- Multi-pet **Book a stay**; online boarding enabled by default on upgrade.
- Boarding compliance, waivers, and vaccination upload fixes.
- Hub REST and booking wizard namespace alignment with KennelFlow Boarding.

## Admin

- KennelFlow Hub home for front-desk staff without `manage_options`.
- Staff Permissions matrix REST API; demo data sandbox; revenue and webhook settings.

See `readme.txt` changelog for the full version-by-version list (0.2.4–0.3.23).
